🗂 SCHEMA SANITY — competitionResult

Pola:

* title (required)
* slug
* date (required)
* city (required)
* country (default: Polska)
* medals:

  * gold
  * silver
  * bronze
* description (Portable Text, required)
* isVisible (boolean — publish toggle)
* seo:

  * metaTitle
  * metaDescription

Walidacje:

* wszystkie główne pola wymagane
* medale ≥ 0
* CMS uproszczony (form-only)

---

🔍 GROQ QUERY

Pobierane tylko widoczne rekordy:

*[_type == "competitionResult" && isVisible == true]
| order(date desc)

---

⚙ JS — results.js

Logika:

1. Fetch danych z Sanity API
2. Zapis do allResults[]
3. Render batchami (STEP = 3)
4. Portable Text → uproszczone HTML
5. PL format daty via toLocaleDateString
6. Disabled load button przy końcu listy
7. Error fallback UI

Struktura:

* loadResults() → pobranie danych
* renderBatch() → render kolejnej porcji
* createCard() → generowanie markup
* portableToHTML() → minimalny renderer rich text
